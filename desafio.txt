Caso de teste 1: NoSuchElementException

Abrir a página

Clicar no botão Add

Verificar se o campo de input da Linha 2 é exibido

A Linha 2 não aparece imediatamente. Este teste irá falhar com org.openqa.selenium.NoSuchElementException se não houver uma espera adequada (explicit wait ou fluent wait).



Caso de teste 2: ElementNotInteractableException

Abrir a página

Clicar no botão Add

Aguardar o carregamento da segunda linha

Digitar texto no segundo campo de input

Clicar no botão Save usando o locator By.name("Save")

Verificar se o texto foi salvo

Esta página contém dois elementos com o atributo name="Save".
O primeiro está invisível. Ao tentar clicar nesse elemento invisível, ocorre ElementNotInteractableException.

Anteriormente essa ação lançava ElementNotVisibleException, mas agora lança uma exceção diferente (não está claro se é mudança de comportamento do Selenium ou bug).



Caso de teste 3: InvalidElementStateException

Abrir a página

Limpar o campo de input

Digitar texto no campo de input

Verificar se o texto foi alterado

O campo de input está desabilitado.
Tentar limpar um campo desabilitado gera InvalidElementStateException.

É necessário primeiro habilitar a edição clicando no botão Edit.

Se tentarmos digitar texto diretamente no campo desabilitado, receberemos ElementNotInteractableException, como no Caso de teste 2.



Caso de teste 4: StaleElementReferenceException

Abrir a página

Localizar o elemento de texto das instruções

Clicar no botão Add

Verificar se o elemento de instruções não está mais exibido

O elemento de instruções é removido do DOM quando a segunda linha é adicionada.
Por isso não é mais possível interagir com ele. Caso contrário, ocorrerá StaleElementReferenceException.



Caso de teste 5: TimeoutException

Abrir a página

Clicar no botão Add

Aguardar 3 segundos para que o segundo campo de input seja exibido

Verificar se o segundo campo de input está visível

A segunda linha aparece após aproximadamente 5 segundos.
Um timeout de 3 segundos não é suficiente, resultando em TimeoutException durante a execução do teste.
